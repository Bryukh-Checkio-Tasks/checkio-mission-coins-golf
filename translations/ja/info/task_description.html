ニコラは彼の手が届く範囲のもので精神を鍛錬します。 彼は仕事机にいくつかのコインを見つけて、それらを使ったマインドゲームを思いつきました。

あなたはいくつかのコインが与えられます。 それぞれのコインは1から9の価値があります。 あなたは入力としてコインのリストを受け取って、それらを使って支払うことのできない最も少ない 正確な額(0を除く)を計算する関数を書かなくてはいけません。

私たちはこのチャレンジに対してもう一つのルールを課します: これは コードゴルフミッションであなたの主なゴールはあなたのコードを出来る限り短くすることです。 コードを短くすればより多くのポイントを獲得することができます。 あなたのスコアは動的でコードの長さに直接関連しています。 参考までにスコアは使われた文字の数に基づいています。 300 文字が許容される最大文字数でこの時は0点です。 300文字から1文字減るごとに1ポイント獲得することができます。 例えば200文字の長さのコードでは100ポイント獲得することになります。

入力 整数のリストとしてコインの価値が与えられます。

出力 これらのコインを使って支払うことのできない最も小さい額を整数として出力します。

例 

golf([9, 2, 2, 1]) == 6
golf([1, 1, 1, 1]) == 5
golf([1, 2, 3, 4, 5]) == 16

例 これは古典的な組み合わせ問題です。 趣味のマインドゲームとして使ったり、スナックマシーンをプログラムするのに使うことができます。

事前条件
0 < len(coins) ≤ 10
all(0 < c < 10 for c in coins)
